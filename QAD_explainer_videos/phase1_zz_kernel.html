Phase 1 · Fingerprint Check

Every row — one card purchase or one fibre-cable scan — gets a digital fingerprint. The quantum kernel compares each fingerprint to a reference set of known-normal fingerprints; rows very different from every normal reference are flagged as suspicious. Unlike classical methods, the quantum kernel compares how every pair of features relates at the same time — not one at a time.
Important: the pipeline is unsupervised. We do not train on "these are fraud, these are normal." The method must find anomalies from features alone. The red/blue colouring in the video is shown to you only so you can see whether the kernel got it right — the kernel itself is colour-blind.
Credit-card transactions
A "row" = one purchase (amount, time, merchant, device, country…). "Normal R" = known-good purchases. Fraud hides when each field looks plausible alone — the combination (odd amount × new merchant × foreign country at 3 AM) reveals it. The kernel catches combinations in one shot.
Optical-fibre OTDR traces
A "row" = one laser-pulse scan, return-loss at 102 distance bins along the cable. "Normal R" = healthy-fibre scans with their smooth exponential decay. An intrusion (tap, dig, bend, break) creates a correlated spike-plus-loss across neighbouring bins that no single-bin check notices.
Classical
One feature at a time — like checking one ID field or one fibre-bin at a time. Misses patterns hidden across correlated fields.
Quantum Kernel
Entire feature-interaction fingerprint at once: every pair of fields in a transaction, every pair of bins in a scan. Catches correlated anomalies.

Two Real Datasets, Side-by-Side

Credit Card Transactions
Financial payment-card fraud · 284,807 rows · 0.17% fraud · 30 PCA-anonymised features. Each row is one European card purchase (amount, time, merchant fingerprint, etc., PCA-scrambled for privacy). Fraud = confirmed chargeback. Highly imbalanced — the needle-in-haystack setting banks actually face.
Optical-Fibre OTDR Traces
Photonic-fibre return-loss traces · 372 rows · 43% anomaly · 102 channels. Each row is one OTDR scan (Optical Time-Domain Reflectometry): a short laser pulse is sent down a fibre-optic cable and the back-scattered light is recorded at 102 distance bins along the cable. An intact fibre gives a smooth exponential decay; an anomaly — a tap (someone listening in), a dig strike, a sharp bend, or a break — shows up as a spike or dip at the event's position plus subtle losses downstream. Telecoms run these scans continuously to monitor long-haul and submarine cables. Balanced by design (43% anomaly) so classifiers have enough positive examples to learn from.
You'll see the same five stages run in parallel on both datasets. Phase 1 reduces the pool by keeping only the top-scoring suspicious rows and passes them to Phase 2.

Phase 1 · ZZ-Kernel Prescreen

Circuit

Hadamards put each feature register in superposition. RZ(2x_i) encodes single-feature angles. CNOT · RZ(2 x_i x_j) · CNOT encodes feature pairs — the source of the quantum fingerprint. Kernel S_K = 1 − ⟨φ(x), φ(x')⟩_R against a reference set R of known-normal rows.

Reference set R = a small sample treated as "known-normal". The kernel scores every row by distance from R; rows far from every member get high S_K.

Top-k% values differ per dataset — they come from the tuner. The 2026-04-21 convergence tuner picked 1.35% on Credit Card → 3 845 rows enter Phase 2, and 12.5% on OTDR → 46 rows enter Phase 2. Those exact counts are what the "selected → Phase 2" metric shows, and they are handed forward to Phase 2 as the autoencoder training pool.

The visual is a 120-dot sample of each dataset (we cannot draw 284 807 dots). So on screen you will see ~2 dots above the cut on Credit Card and ~15 on OTDR — a visual proxy for the real counts reported in the metrics panel.

Why is fraud% in selected so high? That's the prescreen doing its job: concentration rises sharply from the base rate. A small pool with high fraud density is exactly what Phase 2 wants.
